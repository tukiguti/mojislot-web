# 演出アセットと液晶レイアウト

UI レイアウト（メイン液晶）と、章ごとの演出アセット（カットイン一枚絵）の設計・生成手順をまとめる。

## 液晶レイアウト比率

縦積みレイアウト。体感比率を **演出液晶 : リール : ボタン = 1.3 : 0.9 : 0.8** に調整している。

| 区分 | 寸法（Pixi 内部） | 備考 |
|------|------------------|------|
| canvas 全体 | `600 × 732`（縦長 約1:1.22） | `CANVAS_W` / `CANVAS_H`（`src/main.ts`） |
| 演出液晶（上部・ジン/カットイン/演出） | `600 × 432` | `LIQUID_AREA_H = 432`（= リール300の約1.3倍） |
| リール領域（下部） | `600 × 300` | `CELL_HEIGHT(100) × VISIBLE_CELLS(3)` |
| ボタン帯（DOM） | `--control-h`（固定） | ボタン自体は基準の 0.8 倍 |

CSS 側は `--game-w`（cabinet 幅）と `--game-h = game-w × 1.22`、`--control-h` で構成（`src/style.css` の `:root`）。
比率を変えるときは **main.ts の `LIQUID_AREA_H` / `CANVAS_H`** と **CSS の倍率(1.22) / `#lcd-fx` の height(432/732)** を揃えて変更する。

## 演出は「演出液晶」内にクリップ

全画面 DOM 演出（フラッシュ / 紙吹雪 / プレミアムカットイン / ボーナスのキラキラ / 多重HITバッジ）は
`#game-area` 内の **`#lcd-fx`**（= 演出液晶エリア `height: calc(432/732*100%)`, `overflow:hidden`）に出す。

- `src/ui/Effects.ts` の `setEffectHost(el)` で出力先を差し替え、`main.ts` が起動時に `#lcd-fx` を生成して設定。
- これにより演出が cabinet 外・画面全体へはみ出さない（スマスロの上部液晶イメージ）。
- カットイン文字は `container-type: size` + `cqw` で**液晶幅に追従**（全画面前提のはみ出し防止）。
- 進行ステータス（BONUS残り/連チャン/レスキュー）は `#lcd-status`（演出液晶 左上, z-60）に表示し、ヘッダーを溢れさせない。

## 章ごとのカットイン一枚絵

### 命名規約・参照

- 配置: `public/art/cutin_{章id}.webp`（例: `cutin_katakana_animal.webp`）
- 章id は `src/data/chapters.ts` の `ChapterBundle.id`
- `main.ts` が `chapterCutinUrl = ${BASE_URL}art/cutin_${chapterId}.webp` を組み立て、`showPremiumCutin(name, symbols, url)` に渡す
- 画像が無い章は 404 になり、**カットインは CSS 光線のみ**にフォールバック（壊れない）

### 表示

`.premium-cutin-art`（`src/style.css`）が veil の上・光線/役名テキストの下に敷かれ、出現時に軽くズームイン。
表示枠は演出液晶 `600:432`（≒ 1.389:1 横長）。

### サイズ仕様（重要）

カットイン画像は **演出液晶ピッタリの横長 = 1500×1080（アスペクト 600:432 = 1.389:1）** に統一する。
codex は規定アスペクトでしか生成できないため「生成は横長 → 後処理で確定px」の2段で作る。

## アセット生成手順（codex-imagegen）

`zikken/.claude/skills/codex-imagegen` の手順で codex CLI の `$imagegen`（gpt-image）を使う。

```bash
# 1. 生成（横長で発注、master は assets/art-src/ に保存）
codex exec --sandbox workspace-write \
  "画像生成機能(\$imagegen)を使って … 横長構図(landscape) … 画像内に文字は入れない …
   /Users/…/mojislot-web/assets/art-src/cutin_{章id}.png に保存。" < /dev/null

# 2. 演出液晶比率(1500×1080)へ center-crop → WebP 化
magick assets/art-src/cutin_{章id}.png -resize 1500x1080^ -gravity center -extent 1500x1080 -strip /tmp/n.png
cwebp -q 85 /tmp/n.png -o public/art/cutin_{章id}.webp
```

- 画風: **レトロ・パチスロネオン**（暗黒背景・ネオン縁取り・金/紫/緑系・スターバースト・主役モチーフ中央集約・文字なし）
- master PNG（各3MB前後）は `assets/art-src/`（`.gitignore` 済み・ローカル保持）。デプロイは `public/art/` の WebP（各 ~0.5MB）のみ。

## デバッグ（設定 → デバッグ）

新規演出の確認用ボタンを `src/ui/SettingsOverlay.ts`（`DebugActions`）+ `main.ts` に用意。

- **カットイン**: 現章のプレミアム役＋章カットイン画像で強制発動
- **狙え！予告**: aim 演出を強制発動
- 既存: BONUS突入 / 示唆 / クイズ / テンパイSE / 役成立 / 全画面FX

## 保留中

- **body 章背景**（`bg_{章id}.webp` を画面周囲に敷く）は一旦オフ。
  `main.ts` で `has-chapter-bg` クラスを付与しないだけで、CSS（`body.has-chapter-bg`）と `bg_security.webp` は復活用に残置。
