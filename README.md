# mojislot-web

3文字の日本語（ひらがな/カタカナ）を作り出すパチスロ風ゲームの **Web プロトタイプ**。目押し（ビタ押し）で役を狙う、**技術介入が主軸**のゲーム。

---

## 文字スロ ―MOJISLOT―

### 読めるか。狙えるか。**言葉で獲る、技術介入AT。**

**設定は、無い。あるのは、腕だけ。**

#### ▶ ボーナスを引け。目押しを決めろ。ゲーム数を上乗せしろ。

**通常時は、増えない。**
掛け枚数と払い出しが同額。淡々と回し、**演出が降りてくるのを待つ**。

演出は**すべて本物**だ。ガセは一枚も入っていない。
液晶が動いたその瞬間だけが、あなたの取り分。**出れば獲れる。出なければ、獲れない。**

赤が閃けばRB、金が輝けばBB。**目押しを、決めろ。**

**――ボーナス突入。ここからが本番だ。**

ボーナス中は**毎ゲーム演出**。小役を**連続で揃え続けろ**。
連チャンを切らさなければ、1枚の重みは膨れ上がる。**6枚 → 9 → 12 → 18 → 24 → 30枚。最大×10倍。**
だが、**一度でも外せば、ゼロから。**

そして――ボーナス中にもう一度、**赤が来る。金が来る。**

**そいつを目押しで揃えろ。ゲーム数が上乗せされる。**

**上乗せに、上限は無い。**
繋げた分だけ、ボーナスは終わらない。

> 「どれかな」と考えて、打つ。
> それが、このスロットの全てだ。

**― ツキグチ機工 ―**

## ゲームフロー

```
通常時（増えない）
  └─ 演出（発生率50%・全て引き込みあり）
       ├─ クイズ 25% … 出題される。答えの単語を自分で考え、その3文字を狙う
       ├─ 示唆   15% … 青→黄→緑（小役）／赤（RB）／金（BB）
       └─ 狙え！ 10% … 予告された3文字を狙う
            │
            └─ 赤/金示唆・BB/RB予告を目押しで揃える ──▶ ボーナス突入
                                                          │
ボーナス中（演出100%・出玉の山）◀────────────────────────┘
  ├─ 小役を連続で揃える → コンボ倍率が上昇（6→9→12→18→24→30枚）
  └─ 赤/金示唆・クイズでBB/RBを揃える → ゲーム数上乗せ（おかわり・上限なし）
```

## 特徴

- **5章**（ひらがな食べ物 / カタカナ動物 / カタカナ野菜 / ひらがな動詞 / IT・セキュリティ）。役もクイズも図柄も章ごとに総入れ替え
- 役駆動設計：1章あたり コア4役＋チェリー＋RB＋プレミアム2（7揃い／バー揃い）
- **クイズ 105問**（各章21問）。答えの単語を自分で思い出し、その文字を目押しで狙う
- **BIG/REG 二段ボーナス**。出玉の山はボーナス中の **コンボ（連チャン）** と **ゲーム数上乗せ** に集約
- **ガセ演出なし**：演出はすべて引き込みを伴う。裏返しとして**演出なしのスピンは小役も蹴られる**（`assist.kickCore`）＝「演出中に目押しできた時だけ獲れる」
- **モーションブラー**：回転中の図柄に縦方向の残像を速度比例でかける。これにより**実機速度域（24〜28コマ/秒）でも図柄を追える**（ブラーが無いとコマ送りに見えてカクつくため、以前は20コマ/秒に落としていた）。停止した瞬間に0へ戻すので出目はくっきり読める
- **見やすさは自分で決める**：**リール速度（20〜28コマ/秒）とブラー強さ（0.00〜1.00）を設定のスライダーで調整**。どちらもドラッグ中に即反映されるので、回しながら好みの値を探せる（滑らかさ ⇄ 目押しのしやすさのトレードオフ）
- 出現確率・演出レート・目押し補助・予告の中身（BB/RB比率）・リール速度・ブラー既定値は **`data/tuning/default.json`**、配当は **`data/payouts/default.json`** に集約（データ駆動）
- 計数履歴にはルール版、使用速度、AUTO使用、ミッション、デバッグ有無を保存。旧履歴は条件不明として互換表示

## 技術スタック

- **Vite** + **TypeScript**
- **Pixi.js v8**（ゲーム画面のレンダリング）
- **DOM + CSS**（UI レイヤ・演出オーバーレイ）
- **Zod**（JSONスキーマバリデーション）
- **LocalStorage**（セーブ／カード）
- **Vitest**（ユニットテスト）

## 開発

```bash
npm install
npm run dev      # 開発サーバ
npm run build    # 本番ビルド（tsc + vite）
npm run preview  # ビルド成果物のローカル確認
npm test         # Vitest によるユニットテスト
```

## ディレクトリ構成

```
src/
├── main.ts          # エントリーポイント（Pixi 初期化 + DOM 接続・ゲーム進行）
├── core/            # 描画非依存のゲームロジック（ReelEngine/YakuJudge/PayoutCalc/Paylines）
├── productions/     # 演出ロジック（EffectScheduler/BonusZone/SlipResolver/TenpaiDetector 等）
├── render/          # Pixi 描画層（ReelView/EffectVisual/JinView 等）
├── ui/              # DOM UI 層（Effects/SettingsOverlay/ZukanOverlay 等）
├── card/            # 図鑑・統計・セーブ（カード）のコーデック/管理
├── audio/           # 効果音・BGM（Web Audio 合成）
├── router/          # 画面ルーティング
├── data/            # JSON ロード + Zod 検証（schemas/chapters）
└── lib/             # 汎用ユーティリティ（Observable 等）

data/                # ゲームデータ（JSON・Zod 検証）
├── reels/           # 章ごとのリール配列（21コマ×3）
├── yaku/            # 章ごとの役定義
├── quizzes/         # 章ごとのクイズ（各21問）
├── payouts/         # 配当（default.json）
└── tuning/          # 演出レート・補助・蹴り・予告の重み等（default.json）

public/art/          # 画像（カットイン・図柄・狙え等の webp）
tests/               # Vitest によるユニットテスト
```
